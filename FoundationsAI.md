### 1.1 рҹҺҜ Delegar вҖ” ВҝQuГ© confГӯo a la IA?

**Delegar** no significa entregar todo el trabajo a la IA. Significa **tomar una decisiГіn consciente** sobre quГ© tareas son apropiadas para delegarle y cuГЎles requieren tu juicio humano.

> рҹ’Ў **Concepto Clave:** Delegar es mantener la **conciencia del problema**. TГә decides quГ© parte del trabajo la IA puede asistir, pero nunca pierdes la visiГіn global de lo que estГЎs construyendo o aprendiendo.

#### ВҝQuГ© SГҚ delegar?

| Tarea                             | Ejemplo                                            | ВҝPor quГ© sГӯ?                                  |
| --------------------------------- | -------------------------------------------------- | --------------------------------------------- |
| **GeneraciГіn de boilerplate**     | Crear un CRUD bГЎsico, configuraciones repetitivas  | Ahorra tiempo sin perder comprensiГіn          |
| **BГәsqueda de sintaxis**          | "ВҝCГіmo se usa `Stream.collect()` en Java?"         | Es referencia, no razonamiento                |
| **Refactoring mecГЎnico**          | Convertir un `for` a `Stream`, renombrar variables | La lГіgica no cambia, solo la forma            |
| **GeneraciГіn de tests unitarios** | Crear casos de prueba para un mГ©todo existente     | TГә validarГЎs si los tests son correctos       |
| **DocumentaciГіn de cГіdigo**       | Generar Javadoc o comentarios explicativos         | TГә verificas que la documentaciГіn sea precisa |
| **ExploraciГіn de tecnologГӯas**    | "ExplГӯcame quГ© es WebFlux y cuГЎndo usarlo"         | Como punto de partida, no como verdad final   |

#### ВҝQuГ© NO delegar?

| Tarea                           | ВҝPor quГ© NO?                                                               |
| ------------------------------- | -------------------------------------------------------------------------- |
| **Decisiones arquitectГіnicas**  | Requieren contexto del negocio, equipo y restricciones que la IA desconoce |
| **LГіgica de negocio crГӯtica**   | La IA no entiende las reglas del dominio de tu empresa                     |
| **RevisiГіn de seguridad final** | La IA puede generar cГіdigo vulnerable sin saberlo                          |
| **Entendimiento profundo**      | Si delegas la comprensiГіn, no aprendes nada вҖ” solo copias                  |

### 1.2 рҹ“қ DescripciГіn вҖ” ВҝCГіmo dirijo a la IA?

>[NOTE] рҹ’Ў **Concepto Clave:** La descripciГіn efectiva transforma a la IA de un "generador aleatorio" a un **colaborador enfocado**. Cuanto mejor describes el contexto, las restricciones y el resultado esperado, mГЎs Гәtil es la respuesta.

### 1.3 рҹ”Қ Discernimiento вҖ” ВҝEvalГәo lo que recibo?

**Discernir** es la capacidad de **evaluar crГӯticamente** todo lo que la IA produce. Es, posiblemente, la "D" mГЎs importante, porque sin discernimiento, te conviertes en un simple copista que pega cГіdigo sin entenderlo.

> рҹ’Ў **Concepto Clave:** La IA puede generar respuestas que *suenan* correctas pero estГЎn completamente equivocadas. Esto se conoce como **alucinaciГіn**. El discernimiento es tu escudo contra cГіdigo defectuoso, patrones obsoletos y soluciones que no encajan en tu contexto.

#### ВҝQuГ© evaluar en CADA respuesta de la IA?

| DimensiГіn         | Preguntas Clave                                                                   |
| ----------------- | --------------------------------------------------------------------------------- |
| **CorrecciГіn**    | ВҝEl cГіdigo compila? ВҝLa lГіgica es correcta? ВҝLos tipos son compatibles?           |
| **ActualizaciГіn** | ВҝUsa APIs y versiones vigentes? ВҝO sugiere mГ©todos deprecados?                    |
| **Seguridad**     | ВҝHay SQL injection? ВҝExpone datos sensibles? ВҝManeja autenticaciГіn correctamente? |
| **Rendimiento**   | ВҝEs eficiente? ВҝTiene problemas O(nВІ) ocultos? ВҝManeja memoria correctamente?     |
| **Contexto**      | ВҝSe ajusta a MI arquitectura y stack? ВҝSigue las convenciones de MI equipo?       |
